Hexo+github page搭建博客

前言

前前后后大约一个星期的课余时间，借鉴许多大神的文章，终于给博客搭建好了，虽然还不是那么的完善，但是也是做够了，后续抽空再包装一下。第一篇博客就纪录自己博客搭建的过程吧！

简单介绍

Hexo

正如hexo的官方文档所说：Hexo，是一个快速、简洁且高效的博客框架。
Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

github

这个应该都会知道的，全球最大的同性网站。不知道的话去注册一个，然后在Explore栏目下有Resource，
这里面有许多文档比如github guides，十分详细的介绍了有关github操作的知识

github page

在github guides里面详细介绍了，专门为技术人员写博客用的，也能够绑定自己的域名，后续会介绍的

所需软件

本文只限于window系统
git
node.js
IDE,集成开发环境，像sublime，webstrom，intellij idea，网上都能搜得到，以下用的是sublime

搭建之前的建议：学会看官方文档

hexo
github
github page

正式搭建

git的安装

下载安装包，即可安装，由于国内网络问题，需要多试几次，记住网络必须良好，安装结束，桌面右击会多出两个，Git GUI Here,Git Bash Here,点击Git Bash Here,输入git --version,出现下载的版本即可成功。

node.js的安装

直接安装，安装结束，打开Git Bash，输入npm -v,出现下载的版本即可成功，还是由于网络问题，我们需要用淘宝提供的源，复制下面一段代码

然后检验淘宝源是否生效，输入cnpm info express，如果出现一大段的代码即是成功，如果没有则为失败，需要重新的输入，淘宝源只是一次性的，每次重新打开需要复制以上的代码。
以上都是hexo搭建所需要的环境，务必将以上几个弄熟悉再安装hexo，重点熟悉github 和 git常用的命令

安装Hexo

直接安装

先看官方文档，了解hexo的基本情况，上面有链接。因为用的是淘宝源，所以cnpm install -g hexo-cli，会出现一大段安装的东西，当然需要卸载的话直接npm uninstall -g hexo-cli
接着在E盘建一个hexo的文件夹，然后在Git Bash中cd e:/hexo,回车执行hexo init，打开你的hexo会出现这么几个文件

  • 以后进入hexo，直接点击hexo文件夹然后右击Git Bash即可进入
  • 启动hexo本地服务，看看默认的博客是怎么样的，执行hexo server
  • 浏览器访问http://localhost:4000/
  • Ctrl+C即可停止本地服务

更换主题

默认主题太普通，我们来更换next主题，当然以后你想换主题直接这样就行。
看官方文档，看官方文档，看官方文档 点击进入
复制以下命令$ git clone https://github.com/iissnan/hexo-theme-next themes/next等待安装
安装结束，进入本地主站的hexo打开_config.yml，不是theme里面的_config.yml,将里面的theme的默认的landscape改为next，注意：要有空格的
修改完成后，重新在Git Bash输入

  • 重新生成博客的所有内容：hexo generate
  • 重新启动本地服务：hexo server
  • 重新访问：http://localhost:4000/即可看到结果

如果你觉的这种样式不好看，next有三种主题，打开本地hexo中的theme中的_config.yml文件，在sublime直接Ctrl+F查找scheme,大约在115行, 你会看到有三行 scheme 的配置，将你需用启用的 scheme 前面注释 # 去除即可，这里选择Mist。然后保存，重新hexo generate hexo server 访问本地，就可以看到跟我的博客一样的了，至于其他的样式，直接看官方文档，再在网上搜索。
至此，hexo博客搭建已经告段落
