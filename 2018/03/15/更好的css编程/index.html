工作流程

保持容器最小化 尽可能少用不必要的div

少用hack方式 容易出现bug，不易理解

使用css常量 在敏捷开发中使用，比如创建颜色词汇表

使用通用的命名系统 命名规范

根据class和id的语义命名 避免表象方面的命名，在后期js可能会进行调整

利用css级联特性 将共同样式一起编写，各自特性分别编写，减少代码量

使用使用标签 比如small em strong 等等，语义化更加明显

使用简写标记

缩写16进制颜色标记 比如#000代替#000000

根据HATE方法排序伪类链接 Link Visited Hover Active

使用缩写属性 使用缩写减少代码量，但是在特殊时候，具体的代码更加容易理解(顺时针写)

设置字体

在body中字号为62.5% 从而像使用px一样使用em 1em=16px 当然em的内容在下面会提及到的，Chrome中默认最小字体为12px

使用css切换大小写 直接使用 text-transform

使用通用字体类型覆盖通用设置 直接使用 font-family

将line-height设置为1.4em – 1.6em。 “line-height:1.4”为易读的行，合理的行高避免了行超过10个字而过长，而且颜色对比也不会太不明显。比如，对于过亮的CRT显示屏来说，纯黑色在纯白色的背景下往往对比太强烈，因此尝试运用米白色(#fafafa是一个好的选择)和暗灰色（#333333也不错）。

设置 html-元素为100.01%。 这个针对字号的奇怪100.01%会对几个浏览器bug进行弥补。首先，设置一个默认的body百分比字号（而不是em）消除了IE/Win系统中的字体缩放比例问题，即使之后在其它元素中设置为em。此外，一些版本的Opera浏览器会设置一个比其他浏览器小的100%的默认字号。另一方面，Safari浏览器却有一个101%字号的问题。目前流行的最佳解决方案是针对这个属性设置为100.01%的值。”
